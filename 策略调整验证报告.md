
# 策略调整验证报告 (Validation Report)

## 📊 回测结果 (2024.9 - 2026.2)

我们根据之前的发现，将策略从 **"动量 (Momentum)"** 调整为 **"反转 (Reversion)"** (对 15-20日涨幅给予负权重)。

**结果非常成功：**

*   **收益率 (Return)**: **70.72%** (同期基准大幅震荡)
*   **最大回撤 (MaxDD)**: **15.10%** (控制在安全范围内)
*   **夏普比率 (Sharpe)**: **1.61** (非常优秀)

这验证了我们的核心假设：**当前 A 股市场更适合"低吸/反转"而非"追涨"。**

## 🛠️ 代码修复与改进

为了确保实盘和回测能顺利运行，我们修复了以下关键问题：

1.  **`main.py` 修复**：
    *   **缺少 Schedule**: 之前的代码缺少 `schedule(algo, ...)` 调用，导致策略根本不会运行。已修复。
    *   **缺少 Init**: 缺少 `context.br_history` 和 Meta-Gate 阈值变量的初始化，导致回测报错。已修复。
    *   **模式兼容**: 修正了 `init` 中强制覆盖 `mode=LIVE` 的问题，现在可以正确支持回测。

2.  **`core/signal.py` 调整**：
    *   实施了新的权重逻辑：`{1: 30, 3: -20, 15: -20, 20: -30}`。
    *   移除了 `MIN_SCORE` 的硬性过滤（配合 Config 调整）。

3.  **`config.py` 调整**：
    *   **MIN_SCORE**: 从 `20` 调整为 `-50`。这是因为反转策略的得分通常较低（负分），如果不调低阈值，会导致一只股票都选不出来。
